## Project Overview

CrossFit Roppongi website — bilingual JP/EN static site, crossfitroppongi.com.
Stack: Plain HTML + Vanilla JS + Vite, hosted on Firebase Hosting.
Backend: Google Apps Script Web App (Calendar API, Square API, Sheets, GmailApp).
Legacy Firebase Functions (`functions/`) remain for `/api/**` routes during transition.

## Development Commands

```bash
# Install dependencies (run from project root)
npm install
cd functions && npm install

# ── Frontend dev (Vite HMR at http://localhost:5173) ──────────────────────
npm run dev

# ── Backend dev (Firebase Functions emulator at http://127.0.0.1:5001) ───
firebase emulators:start --only functions --project crossfitwebsite

# Vite proxies /api/* → http://127.0.0.1:5001/crossfitwebsite/us-central1/apiV2
# Run both terminals simultaneously for full local stack.

# ── Build & preview ───────────────────────────────────────────────────────
npm run build          # outputs to dist/
npm run preview        # serve dist/ locally

# ── Lint functions ────────────────────────────────────────────────────────
cd functions && npm run lint

# ── Deploy ────────────────────────────────────────────────────────────────
# Always build first, then deploy from master branch only
npm run build && firebase deploy
firebase deploy --only hosting   # hosting only (serves dist/)
firebase deploy --only functions # functions only
```

## Architecture

### Frontend
- Source: HTML pages in project root, assets in `assets/`
- Build output: `dist/` (generated by Vite — do not edit, do not commit)
- Styles: `assets/css/styles.css` and `assets/css/media-queries.css`
- Legacy styles also in `master_styles.css` (root) — being phased out
- Images: `assets/img/`
- Icons: PNG format in `assets/images/icons/`
- Uses jQuery (CDN), Swiper.js (CDN), mb.YTPlayer (CDN) for existing pages
- New booking pages: Vanilla JS only, no jQuery

### Vite Config (`vite.config.js`)
- MPA (multi-page) mode — all HTML files listed as entry points
- Add new pages to `build.rollupOptions.input` when created
- Dev proxy: `/api` → Firebase Functions emulator

### Backend — Google Apps Script (primary, new pages)
- All server-side logic for new booking pages: Calendar availability, Square API
  (Customers, Orders), Sheets writes, GmailApp emails
- Square Access Token lives in Apps Script only — NEVER in frontend or .env
- Deployed as Web App; URL stored in `assets/js/config.js` (not sensitive)

### Backend — Firebase Functions (`functions/`) — legacy
- Single Express app exported as `apiV2`
- Routes:
  - `GET /api/square/config` — Square payment config + pricing
  - `POST /api/square/bookings` — Payment processing
  - `POST /api/contact/submit` — Contact form via Formspree

### Environment Configuration
`functions/.env` (see `functions/.env.example`):
- `SQUARE_ACCESS_TOKEN`, `SQUARE_ENVIRONMENT`, `SQUARE_LOCATION_ID`
- `SPARTAN_ID`, `HYROX_PERFORMANCE_ID`, `HYROX_STRENGHT_ID`, `TRIAL_ID`, `DROPIN_ID`
- `BOOKING_NOTIFICATION_WEBHOOK`, `CONTACT_FORM_URL`

## Booking System Rules
- Booking ID format: `CFR-{YEAR}-{5-digit sequential}`
- Hold timer: 15 minutes
- Cancellation: non-refundable; 48-hour change window; one change only
- Drop In 3-Pack credits expire 90 days; valid for Drop In + Open Gym
- Customer identity: Square Customers API lookup by email/phone — no login

## Forms

### Contact Form (`contact.html`)
- Submits via `POST /api/contact/submit` (Firebase Functions → Formspree)
- Fields: inquiryType (select), name, phone, email, message, acceptedPrivacy

### Career Application Form (`career.html`)
- Submits via `POST` to Google Apps Script Web App (URL: `CAREER_FORM_URL` constant in career.html JS)
- Fields: position (select), name, birthday (date), gender (radio), phone, email, file (optional, max 10MB, base64-encoded in JSON), message (optional), acceptedPrivacy
- Position options: `コーチ・トレーナー` | `フロント・フロントマネジャー`
- File encoded as base64 in JSON payload: `{ fileBase64, fileName, fileType }`
- APPLY buttons use `data-position` attribute to pre-select job type in form

**Auto-reply to applicant**

- Attach file if provided
